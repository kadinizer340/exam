Formatting rules

Make the timetable files easy for the backend to read.

  • Use `.xls` or `.xlsx` only.
  • Each sheet tab should match the branch mapping used in `app.py`, such as `cs` or `me`.
  • Every sheet must contain `date` and `subject` columns exactly.
  • Use real Excel date cells whenever possible so the date conversion logic succeeds.
  • The timetable sheets are later mapped back to student sections during seat generation.
